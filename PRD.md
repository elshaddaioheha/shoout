SHOOUTS – TECHNICAL & PRODUCT
SUMMARY (FOR DEVELOPERS) 
 
1. What Shoouts Is (High-Level) 
Shoouts is an African-first digital music platform that combines: 
● A music marketplace (buying & selling music IP)
● A secure, creative storage system
● Localized African payments
● Creator-focused monetization tools 
The platform is built to help artists, producers, songwriters, and music executives monetize,
protect, and share their intellectual property—without relying on Western-centric platforms. 
 
2. Core Product Architecture (3-Mode System) 
Shoouts has three integrated product modes: 
A. Shoouts Vault (Storage Mode) 
Purpose: Trust, privacy, retention 
Key functions: 
● Upload & store audio files
● Folder-based organization
● Private & shareable links
● Download / listener tracking
● File permissions (view-only / download) 
This is built first in the MVP. 
 
B. Shoouts Studio (Marketplace Mode) 
Purpose: Monetization 
Key functions: 
● List beats, songs, and music components for sale
● Set prices & licenses
● Buyer–seller chat
● Payments & payouts
● Automatic file delivery after purchase
● Platform takes transaction service charge 
Built after Vault, once trust is established. 
 
C. Shoouts Hybrid Creator Mode (Premium) 
Purpose: High-value users & revenue 
Key functions: 
● Combines Studio + Vault
● Sell directly from stored files
● Unified dashboard (storage + sales)
● Subscription-based access
● Higher visibility & analytics 
Drives ARPU and long-term revenue. 
 
3. Target Users (Who We Are Building For) 
● Independent artists
● Producers & beat makers
● Songwriters
● Music executives & labels
● African creatives (primary)
● Global users (secondary) 
 
4. Monetization Logic (Important for Dev Decisions) 
Revenue sources: 
1. Subscriptions
○ Studio plans
○ Vault plans
○ Hybrid Creator plans 
2. Transaction fees
○ % retained from every sale 
3. Advertising & promotions
○ Featured listings (later phase) 
Platform currency: USD (reference)
Payments & payouts: Localized (NGN, GHS, etc.) 
 
5. MVP Build Order (Critical) 
Phase 1 (Foundation) 
● Auth & user roles
● Profile setup
● Dashboard shell 
Phase 2 (Vault MVP) 
● File upload & storage
● Sharing & permissions
● Basic analytics 
Phase 3 (Studio MVP) 
● Listings
● Checkout
● Chat
● Transaction handling 
Phase 4 (Hybrid Mode) 
● Vault ↔ Studio integration
● Subscription enforcement
● Unified analytics 
 
6. Non-Negotiables (Engineering Priorities) 
● Secure file storage (encryption)
● Role-based access control
● Scalable architecture
● Low-bandwidth optimization
● Clean audit logs
● Admin moderation tools 
 
7. Success Metrics (What We Track Early) 
● Upload frequency
● File sharing activity
● Paid conversion rate
● Marketplace transactions
● Retention of Hybrid users 
 
8. Long-Term Vision (So Devs Know This Will Scale) 
Post-MVP features: 
● Licensing templates
● Ownership timestamping
● Label & team accounts
● Advanced analytics
● API access 
Shoouts is being built as long-term infrastructure, not a short-term app. 
 
9. One-Line Summary for Devs 
“We are building an African-first music infrastructure platform that starts with secure
storage, adds monetization, and scales through premium creator workflows.”
