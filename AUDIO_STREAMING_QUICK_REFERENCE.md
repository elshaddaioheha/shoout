# Audio Streaming Quick Reference

## Three-Layer Anti-Piracy Model

```
Layer 1: WATERMARK
├─ Audio watermark overlay on previews
├─ "Shoouts!" voice every 15 seconds
├─ Inaudible but unmistakable to pirates
└─ Configured in Mux Dashboard

Layer 2: HLS TRANSCODING (Chunked Streaming)
├─ 6-second chunks instead of full file download
├─ Adaptive bitrate: 128kbps (3G) → 256kbps (WiFi)
├─ Cannot right-click save (not a single file)
└─ Reduces egress costs 80%

Layer 3: SIGNED URLS (Time-Limited Downloads)
├─ 15-minute expiry for verified purchases
├─ 1-hour expiry for marketplace previews
├─ Purchase verification required for full-quality
└─ Prevents sharing between users
```

---

## Storage Structure (Anti-Piracy Optimized)

```
originals/{userId}/{trackId}.wav
  ├─ Raw 50MB original file
  ├─ Backend-only access (NO client direct read)
  └─ 15-min signed URL only after purchase verification

hls-previews/{userId}/{trackId}/
  ├─ manifest.m3u8 (watermarked HLS playlist)
  ├─ segment-000.ts, segment-001.ts... (6-sec chunks)
  ├─ Generated by Mux on upload
  └─ 1-hour signed URL (marketplace preview only)

vaults/{userId}/{filename}
  ├─ Temporary upload staging area
  ├─ Moved to originals/ after processing
  └─ User can read/write own files

system/bestSellers.json
  ├─ Pre-computed top 12 tracks
  ├─ Read-only for clients
  └─ Updated hourly via Cloud Scheduler
```

## Cloud Functions: New Endpoints

### 1. `getStreamingUrl()` - Callable Function

**Purpose:** Return marketplace preview OR library download URL (with security checks)

**TypeScript:**
```typescript
const streamingUrl = await httpsCallable(functions, 'getStreamingUrl')({
  trackId: 'sunset-vibes-123',
  uploaderId: 'producer1',
  isLibraryAccess: false  // true = verify purchase first
});

// Response (Marketplace):
// {
//   url: "https://storage.googleapis.com/...&expiry=...",
//   type: 'watermarked-hls',
//   expiresIn: 3600,  // 1 hour
//   mimeType: 'application/vnd.apple.mpegurl'
// }

// Response (Library):
// {
//   url: "https://storage.googleapis.com/...&expiry=...",
//   type: 'original',
//   expiresIn: 900,  // 15 minutes
//   mimeType: 'audio/wav'
// }
```

**Security:**
- Requires Firebase Auth
- If `isLibraryAccess=true`: Checks `users/{uid}/purchases/{trackId}` exists
- Rejects if no purchase doc found

---

### 2. `processAudioUpload()` - Storage Trigger

**Trigger:** When file uploaded to `vaults/{userId}/{filename}`

**Workflow:**
```
User uploads → vaults/{userId}/track.wav
   ↓
Storage.onFinalize() triggered
   ↓
1. Copy to originals/{userId}/{trackId}.wav
2. Create Firestore doc: uploads/{trackId} = { transcodingStatus: 'pending' }
3. Call initiateHlsTranscoding()
   ↓
*Async: Mux processes and webhooks back*
   ↓
4. onHlsTranscodingComplete() updates: transcodingStatus = 'complete'
```

**Do NOT call directly** — automatic on storage upload.

---

### 3. `onHlsTranscodingComplete()` - Mux Webhook

**Trigger:** Mux API sends `POST` to webhook when transcoding finishes

**Setup (One-time):**
```bash
# In Mux Dashboard:
1. Settings → Webhooks
2. Add Webhook:
   URL: https://[REGION]-[PROJECT].cloudfunctions.net/onHlsTranscodingComplete
   Events: video.asset.ready
   Signing Secret: (copy and store in Cloud Secret Manager)
```

**What it does:**
- Receives notification from Mux
- Finds upload doc by `muxAssetId`
- Updates Firestore: `{ transcodingStatus: 'complete', hlsUrl, playbackId }`

## Cloud Storage Rules Changes

### Strictly Enforced: No Client Direct Read

**OLD (Vulnerable):**
```
match /users/{uid}/audio/{filename} {
  allow read: if isAuthenticated();  // ❌ ANYONE can read
}
```

**NEW (Secure):**
```
match /originals/{userId}/{trackId}.wav {
  allow read: if false;  // ❌ NO CLIENT READ
  // Access only via getStreamingUrl() signed URLs
}

match /hls-previews/{userId}/{trackId}/{segment=**} {
  allow read: if false;  // ❌ NO CLIENT READ
  // Access only via signed URLs (1-hour expiry)
}

match /vaults/{userId}/{filename} {
  allow read: if isOwner(userId);  // User can verify upload
  allow create: if isOwner(userId) && isAudio() && maxSize(200);
}
```

## Client Implementation

### Audio Player Setup (HLS Support)

**Install HLS.js:**
```bash
npm install hls.js
```

**React Native Example:**
```typescript
import HLS from 'hls.js';

function AudioPlayer({ trackId, uploaderId, isPurchased }) {
  useEffect(() => {
    (async () => {
      // Get signed URL
      const result = await httpsCallable(functions, 'getStreamingUrl')({
        trackId,
        uploaderId,
        isLibraryAccess: isPurchased
      });

      // For HLS (.m3u8)
      if (result.data.type === 'watermarked-hls') {
        const hls = new HLS();
        hls.loadSource(result.data.url);
        hls.attachMedia(document.querySelector('audio'));
      }
      // For direct audio file (.wav)
      else {
        document.querySelector('audio').src = result.data.url;
      }
    })();
  }, [trackId, uploaderId, isPurchased]);

  return <audio controls />;
}
```

### Upload Changes (Use vaults/)

**OLD:**
```typescript
await bucket().ref(`users/${userId}/audio/${fileName}`).put(file);
```

**NEW:**
```typescript
await bucket().ref(`vaults/${userId}/${fileName}`).put(file);
// Cloud Functions automatically:
// 1. Copy to originals/
// 2. Transcode to HLS (Mux)
// 3. Update Firestore with streaming URLs
```

### Library Download Button

```typescript
async function handleDownload(trackId: string) {
  try {
    const result = await httpsCallable(functions, 'getStreamingUrl')({
      trackId,
      uploaderId: track.uploaderId,
      isLibraryAccess: true  // Verify purchase
    });

    // Start download (expires in 15 minutes)
    window.location.href = result.data.url;
  } catch (error) {
    if (error.code === 'permission-denied') {
      Alert.alert('No Purchase', 'You must purchase this track to download.');
    }
  }
}
```

## Firestore Security (New Uploads Collection)

**Firestore Path:** `uploads/{trackId}`

```typescript
{
  userId: 'user123',
  fileName: 'sunset-vibes.wav',
  originalStoragePath: 'originals/user123/sunset-vibes.wav',
  hlsManifestPath: 'hls-previews/user123/sunset-vibes/manifest.m3u8',
  
  // Transcoding Status
  transcodingStatus: 'complete',  // 'pending' | 'processing' | 'complete' | 'failed'
  muxAssetId: 'lw23i90df',        // Mux API asset ID
  
  // Streaming URLs (set after transcoding complete)
  hlsUrl: 'https://image.mux.com/lw23i90df/manifest.m3u8',
  playbackId: 'lw23i90df',
  
  // Timestamps
  createdAt: 2024-01-15T10:30:00Z,
  completedAt: 2024-01-15T10:35:00Z
}
```

**Firestore Rules:**
```
// Clients can read uploads metadata (for progress tracking)
match /uploads/{uploadId} {
  allow read: if request.auth != null;
  allow write: if false;  // Backend only
}

// Users collection: list user's uploads (for library)
match /users/{uid}/library/{trackId} {
  allow read: if request.auth.uid == uid;
  allow write: if false;  // Backend only (payment)
}
```

## Cost Comparison

### Storage: Before vs After

```
1,000 tracks @ 5MB average:

BEFORE (Progressive Download):
  - 1,000 × 50MB = 50GB total
  - 50,000 streams × 50MB = 2,500GB egress
  Cost: 50GB × $0.02 + 2,500GB × $0.12 = $1 + $300 = $301/month

AFTER (HLS + Watermark):
  - 1,000 × (50MB original + 100MB HLS chunks) = 150GB total
  - 50,000 streams × 3MB avg = 150GB egress
  Cost: 150GB × $0.02 + 150GB × $0.12 = $3 + $18 = $21/month

SAVINGS: $280/month (~93% reduction)
```

## Monitoring & Alerts

### Key Metrics to Track

```
1. Transcoding Success Rate
   - Average time: < 1 minute
   - Failure rate: < 1%

2. Storage Costs
   - originals/ total size
   - hls-previews/ total size
   - Trend: should stabilize after initial migration

3. Download Verification
   - Requests with isLibraryAccess=true
   - Rejection rate: verify purchase enforcement working

4. Signed URL Usage
   - Average URL lifespan before expiry
   - Downloads that fail due to expired URLs
```

### Sample Alerts (Firebase Monitoring)

```yaml
- Alert: transcodingStatus = 'failed' for > 5 docs
  Action: Check Mux API health + Cloud Function logs

- Alert: hls-previews storage > 500GB
  Action: Delete old (30+ days) previews to free space

- Alert: getStreamingUrl permission-denied > 10%
  Action: Check purchase verification logic
```

```
   firebase functions:delete processAudioUpload

2. SWITCH BACK: Update cart.tsx to use old audio player
   (play from users/{uid}/audio/ path instead of hls-previews/)

3. REVERT: firebase deploy --only storage
   (restore old storage.rules without originals/hls/vaults paths)

4. Wait for investigation before re-enabling

Total rollback time: ~5 minutes
```
